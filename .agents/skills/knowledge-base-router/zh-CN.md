# Knowledge Base Router（知识库路由）

## 中文行为说明

`knowledge-base-router` 是项目知识库体系的唯一公共入口。本文件补充中文场景下的行为指引，不重复 `SKILL.md` 的完整规则。

### 中文触发信号

| 用户意图 | 典型中文表达 |
| -------- | ------------ |
| 初始化 | "初始化知识库"、"创建知识库结构"、"搭一下知识库" |
| 查询 | "查一下"、"有没有关于 X 的文档"、"知识库里怎么说的" |
| 写草稿 | "整理成知识库正文"、"写一篇知识库文档"、"把这段总结放进知识库" |
| 发布 | "保存到知识库"、"发布这篇正文"、"写入知识库" |
| 审计 | "检查知识库"、"哪里不对劲"、"看一下知识库状态" |
| 治理 | "修复索引"、"去重"、"整理知识库"、"清理一下" |

### 中文澄清话术

- Empty + 读写请求："知识库尚未初始化。需要现在初始化吗？"
- Empty + "知识库"（无操作类型）："你想对知识库做什么？比如：初始化、查询、写入、检查？"
- 分类模糊："你想把这篇文档放到哪个分类下？可用的分类有：[列出 categories]"
- Partial/Broken + 任意请求："知识库当前状态不完整，已转交审计检查。需要先修复才能继续。"
- 缺少正文："你还没提供要保存的正文内容。需要我先生成一篇草稿吗？"

### 中文常见误区

1. "只是查一下，不用检查状态" → 即使只读也需要 Bootstrap Gate，Partial 状态下索引可能不可靠。
2. "我直接看看目录里有什么" → `knowledge-base-context` 通过索引发现文档，不能直接扫描目录。
3. "先写进去再说，索引以后补" → 这正是造成孤儿文档的原因。写入必须走 publisher。
4. "整理知识库" → 不能直接启动 gardener。必须先走 auditor 生成报告并确认范围。
