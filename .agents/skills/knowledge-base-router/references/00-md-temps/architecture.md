# architecture

**必填节**：
- **引言与目标** — 驱动因素、质量目标、利益相关者
- **上下文与范围** — 系统边界、外部接口
- **方案策略** — 关键架构决策和思路
- **构建块视图** — 模块/组件划分与职责（不是目录树）
- **部署视图** — 部署拓扑与基础设施

**可选节**：
- 约束 — 技术/组织/法规硬性限制
- 运行时视图 — 关键场景的时序/交互
- 架构决策记录 — 与方案策略互补的重大决策
- 风险与技术债 — 已知风险和有意承担的技术债

**常见错误**：
- ❌ 只有模块图没有部署图（运行时行为不可见）
- ❌ 构建块视图变成目录树（没有职责描述）
