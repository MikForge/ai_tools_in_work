# Markdown 文档命名规范

## 适用范围

- 本规范仅适用于 `docs/00-project-knowledge-base/**` 下的正文 Markdown 文档。
- `README.md` 仅用于目录索引和分类入口，不参与正文文档命名。
- 目录结构和分类定位以 `.knowledge-base.yml` 为准；本规范只定义文件名规则。

## 标准结构

正文文档默认采用：

`类型-主题-行为.md`

示例：

- `spec-登录-流程.md`
- `design-背包系统-重构.md`
- `test-商城-购买.md`

约束：

- `类型` 必填，表示文档类别。
- `主题` 必填，表示稳定对象。
- `行为` 必填，表示动作、切面或用途。
- 扩展名固定为 `.md`。

## 命名生成规则

命名优先级：

1. 用户或调用方显式提供完整文件名时，优先使用。
2. 未提供完整文件名，但外部参数指定了命名语言或候选 `类型`、`主题`、`行为` 时，按参数生成。
3. 两者都没有时，按默认规则生成。

默认规则：

- 默认允许中文文件名。
- 英文模式下，`主题` 转为英文 slug，`类型` 和 `行为` 使用受控英文标识。
- `主题` 应表达稳定对象，如模块、系统、功能域、资源对象或业务对象。
- `行为` 应表达该文档围绕主题讨论的动作、切面或用途。
- 若文档没有明显动作，`行为` 默认使用 `总览`。

常用 `类型`：

- `requirement`
- `spec`
- `design`
- `architecture`
- `solution`
- `guide`
- `test`
- `verification`

常用 `行为`：

- `总览`
- `流程`
- `设计`
- `实现`
- `重构`
- `接入`
- `配置`
- `说明`
- `验证`
- `测试`
- `迁移`

若现有高频项无法准确表达，可以扩展新的稳定词，但不要为一次性表述临时造词。

## 例外、重名与迁移规则

- `README.md` 是保留文件名，只用于目录索引。
- 目标目录下不存在同名文件时，直接创建。
- 目标目录下已存在同名文件时，先判断是更新已有文档还是新增并列文档，不自动生成随机新名。
- 新增并列文档时，可在 `主题` 或 `行为` 中加入最小必要区分信息，但仍保持 `类型-主题-行为.md` 结构。

示例：

- `spec-登录V2-流程.md`
- `solution-背包系统客户端-迁移.md`

迁移口径：

- 新建文档必须遵守本规范。
- 仅做 `index-only` 或仅重建索引时，可保留旧文件名。
- 一旦发生内容更新、归档、迁移、重写，应按新规范重命名。

执行原则：

`内容触碰即迁移，索引触碰不迁移。`
