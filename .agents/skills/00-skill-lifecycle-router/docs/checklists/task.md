# 任务输出审查清单

对任务执行输出按以下标准审查。发现问题归类为 严重 / 重要 / 轻微。

## 计划遵循度

- [ ] 每个计划步骤都有对应的输出
- [ ] 无跳过或部分完成的步骤
- [ ] 提交历史与计划任务顺序一致

## 输出质量

- [ ] SKILL.md 包含有效 YAML frontmatter（name, description）
- [ ] SKILL.md 遵循项目 skill 约定（按需包含 Workflow、Rules、Anti-Patterns 章节）
- [ ] 引用的脚本或模板存在且可用

## 模式一致性

- [ ] 新 skill 的结构与已有已验证 skill 模式一致
- [ ] 未在设计文档中明确说明的情况下不引入新架构模式
- [ ] 文件命名和目录结构遵循项目约定
