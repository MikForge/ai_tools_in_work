# 设计文档审查清单

对每份设计文档按以下标准审查。发现问题归类为 严重 / 重要 / 轻微。

## 范围与边界

- [ ] 问题陈述清晰——这个 skill 解决什么问题？
- [ ] 范围明确界定——哪些在范围内、哪些不在
- [ ] 非目标已列出并说明理由

## 一致性

- [ ] 无相互矛盾的断言（如"路由器只做路由" vs "路由器验证内容"）
- [ ] 全文术语统一（同一概念使用同一词汇）
- [ ] 架构图与文字描述一致

## 完整性

- [ ] 无"TBD"或"TODO"占位符
- [ ] 所有引用文件/产物有明确路径
- [ ] 边界情况已处理（空状态、错误状态、边界条件）

## 模式一致性

- [ ] 遵循项目约定（文件命名、目录结构、YAML frontmatter）
- [ ] 不引入与已有已验证模式冲突的模式
- [ ] 如偏离已有模式，有明确的偏离理由

## 模式一致性（来自 spec mode-copy-defense）

- [ ] 设计是否引入了新的结构模式？如是，标记审查
- [ ] 设计与本项目类似 skill 的结构是否一致？
