
# 技能设计约束

## 关联文档/目录引用规则

### 目标

解决 skill 文档中路径引用分散、硬编码、命名不一致的问题。通过统一的引用链接机制，实现路径的单点维护和跨文档一致性。

### 规则

**引用名称** — 使用中文语义名，不使英文 slug：

| 类别 | 引用名格式 | 示例 |
| ---- | ---------- | ---- |
| 子技能 | `[子技能-<阶段>]` | `[子技能-设计]`、`[子技能-反馈]` |
| 约束文档 | `[<约束名>约束文档]` | `[技能约束文档]`、`[测试约束文档]` |
| 产物目录 | `[<产物名>文档目录]` | `[设计文档目录]`、`[评审文档目录]` |

**定义位置** — 所有引用链接的定义统一放在文末 `## 关联文档/目录` 章节下，按子技能、约束文档、产物目录分组。

**正文引用** — 正文中使用简短引用格式 `[引用名]`，不硬编码路径。

### 禁止

- 禁止在正文中硬编码路径（如 `docs/products/specs/`）
- 禁止使用英文 slug 作为引用名（如 `[product-specs-dir]`）
- 禁止将引用定义分散在各处而非统一放在文末 `## 关联文档/目录`

### 示例

```markdown
<!-- 正确：使用引用链接 -->
读取 [评审文档目录] 或 [测试文档目录] 中的报告
扫描目标工作区 [反馈文档目录]<skill-name>-feedback.md

<!-- 错误：硬编码路径 -->
读取 docs/products/reviews/ 或 docs/products/test/ 中的报告
扫描目标工作区 docs/notes/<skill-name>-feedback.md
```
