# 计划文档审查清单

对每份计划文档按以下标准审查。发现问题归类为 严重 / 重要 / 轻微。

## 可执行性
- [ ] 每个步骤可独立执行（不出现"先做 X，再做 Y"这种合并步骤）
- [ ] 每个步骤有精确的文件路径
- [ ] 每个代码步骤展示完整代码（不写"类似任务 N"）
- [ ] 每个命令步骤有预期输出

## 完整性
- [ ] 无"TBD"或"TODO"占位符
- [ ] 所有引用的 spec、文件或依赖存在
- [ ] 每个任务定义了测试策略

## 依赖顺序
- [ ] 任务排序正确——不前向引用尚未定义的类型/函数
- [ ] 文件创建在文件修改之前
- [ ] 测试文件创建在实现之前

## 模式一致性
- [ ] 计划结构匹配项目计划约定
- [ ] 提交信息风格一致
