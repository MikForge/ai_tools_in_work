# 反馈说明格式

每条反馈说明遵循以下结构。一条说明可包含多个问题。

```markdown
# 反馈说明：<skill-名称>

## 问题

### <问题标题>
- **严重度：** 严重 | 重要 | 轻微
- **来源：** review | test | self-check | user
- **回流目标阶段：** design | plan | task | test
- **描述：** <发现什么以及为什么重要>
- **文件：** <路径:行号，如适用>

### <问题标题>
...
```

## 格式规则
- 每个问题必须包含 严重度、来源、回流目标阶段、描述
- 严重度遵循审查标准：严重 = 阻断/安全，重要 = 缺失/错误/不足，轻微 = 风格/优化
- 回流目标阶段决定反馈回流到哪个阶段进行修复
- 对同一 skill 多次触发反馈时，问题追加（不覆盖）
