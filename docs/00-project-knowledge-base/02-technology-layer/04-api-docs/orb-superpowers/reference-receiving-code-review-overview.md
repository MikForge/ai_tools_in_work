# receiving-code-review — 接收代码审查

> 来源: [obra/superpowers/skills/receiving-code-review](https://github.com/obra/superpowers/tree/main/skills/receiving-code-review)

## 概述

在收到代码审查反馈时使用，尤其是反馈看起来不清晰或技术上可疑时——需要技术严谨性和验证，而非表演性同意或盲从。

## 关键设计决策

- **技术评估优先**：代码审查需要技术评估，而非情绪表演
- **禁止模式回应**：严禁 "You're absolutely right!" 等无审慎回应的模式化回答
- **验证前置**：在实施任何建议前必须对照代码库实际验证

## 工作流程

1. 完整阅读反馈不反应 → 2. 用自己的话重述需求 → 3. 对照代码库验证
→ 4. 评估技术正确性 → 5. 技术确认或有理有据的驳回 → 6. 逐一实施并测试
