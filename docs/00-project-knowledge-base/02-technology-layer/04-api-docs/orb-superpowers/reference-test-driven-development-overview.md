# test-driven-development — 测试驱动开发

> 来源: [obra/superpowers/skills/test-driven-development](https://github.com/obra/superpowers/tree/main/skills/test-driven-development)

## 概述

在实施任何功能或 bug 修复之前使用。先写测试，观察失败，再写最小代码通过。

## 关键设计决策

- **铁律**：如果没有观察测试失败，就无法确认测试是否测对了东西
- **违反字面规则即违反精神**：不允许 "skip TDD just this once" 的合理化
- **通常必用**：新功能、bug 修复、重构、行为变更
- **例外需询问**：一次性原型、生成代码、配置文件

## 工作流程

1. 写测试 → 2. 观察失败（RED）→ 3. 写最小代码通过（GREEN）
→ 4. 重构（REFACTOR）
