# brainstorming — 头脑风暴设计

> 来源: [obra/superpowers/skills/brainstorming](https://github.com/obra/superpowers/tree/main/skills/brainstorming)

## 概述

在创意工作前进行设计探索。通过自然协作对话，将想法转化为完整的设计和规格文档。

## 关键设计决策

- **Hard Gate**：在任何实现 skill 被调用、代码编写或项目脚手架之前，必须先完成设计并获得用户批准
- **逐层剥离**：单个项目超过一个 spec 的复杂度时，先分解为子项目再逐个设计
- **先验证后提交**：每个设计阶段完成后即获得用户确认，不累积未验证的假设
- **YAGNI 原则**：严格去除所有不必要的功能
- **永远需要设计**：没有"过于简单无需设计"的例外

## 工作流程

1. 探索项目上下文 → 2. 逐个提问澄清需求 → 3. 提出 2-3 方案 → 4. 按节呈现设计并逐节确认 → 5. 写入设计文档 → 6. 自检 → 7. 用户审阅 → 8. 转入 writing-plans
