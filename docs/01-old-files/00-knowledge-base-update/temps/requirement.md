# requirement

> 需求文档模板。文件名：`requirement-<主题>-<行为>.md`

**区别于** spec：requirement 只描述问题和验收条件，不含方案；spec 含方案和备选。

**必填节**：
- **背景** — 业务/技术上下文
- **问题陈述** — 要解决什么问题
- **目标与非目标** — 含明确排除项
- **功能需求** — 按优先级排
- **验收标准** — 可验证的条件（有数字/布尔值/可见行为）

**可选节**：
- 用户故事/用例 — 何时加：需求涉及用户可见行为
- 非功能需求 — 何时加：涉及性能、安全、合规
- 依赖项 — 何时加：依赖外部系统或上游模块

**常见错误**：
- ❌ 验收标准写成"系统正常运行"（不可验证）
- ✅ 验收标准写成"X 条件下 Y 操作在 200ms 内完成"
- ❌ 只有目标没有非目标（scope creep 防线）
