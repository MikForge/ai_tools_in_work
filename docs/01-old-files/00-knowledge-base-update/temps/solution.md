# solution

> 解决方案模板（轻量）。文件名：`solution-<主题>-<行为>.md`

**区别于** design：solution 只给方案思路和理由，不含详细架构、组件划分、数据流。

**必填节**：
- **问题** — 当前状况与痛点
- **方案** — 方案描述
- **理由** — 为什么是这个方案
- **影响评估** — 影响范围、利弊

**可选节**：
- 备选方案 — 何时加：存在至少 1 个合理替代方案
- 迁移步骤 — 何时加：需要分步执行
- 回滚方案 — 何时加：方案有不可逆操作

**常见错误**：
- ❌ 和 design 不分 — solution 不含详细架构和数据流
- ❌ 问题只写现象不写根因
