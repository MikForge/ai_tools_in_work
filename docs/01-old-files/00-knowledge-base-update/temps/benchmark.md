# benchmark

> 性能基准模板。文件名：`benchmark-<主题>-<行为>.md`

**区别于** test：benchmark 关注性能数据（延迟/吞吐），test 关注功能正确性。

**必填节**：
- **目的** — 基准测试目的和场景
- **方法论** — 测试方法、工具、参数
- **环境** — 硬件/软件/网络（缺环境信息结果无法复现）
- **结果** — 数据（表格/图表）
- **分析** — 结论与解读

**可选节**：
- 原始数据 — 何时加：需要他人复现
- 基线对比 — 何时加：有历史数据比较

**常见错误**：
- ❌ 只有结果没有方法论（无法复现）
- ❌ 环境信息缺失（不同环境结果可能相反）
