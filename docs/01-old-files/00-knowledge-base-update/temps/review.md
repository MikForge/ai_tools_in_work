# review

> 评审记录模板。文件名：`review-<主题>-<行为>.md`

**必填节**：
- **上下文** — 评审范围、涉及的文件/设计
- **总体评价** — 一句话结论（必须与发现一致）
- **发现** — 问题清单，按严重度分级（阻断/重要/建议）

**可选节**：
- 建议 — 何时加：有改进建议而非问题
- 后续项 — 何时加：有需要跟踪但不阻断的项

**常见错误**：
- ❌ 发现清单不标严重度（无法判断哪些必须修）
- ❌ 总体评价和发现矛盾（总评说"基本没问题"但列了 5 个阻断项）
