# Markdown 文档命名规范

## 适用范围

本规范定义 Markdown 正文文档的文件命名规则。目录结构、分类体系、索引格式由各自领域自行约定，不在本规范范围内。

## 标准结构

正文文档采用三段式命名：

`类型-主题-行为.md`

- `类型` — 必填，文档类别，从受控词汇表选取。
- `主题` — 必填，稳定对象（模块、系统、功能域等）。
- `行为` — 必填，动作、切面或用途。
- 扩展名固定为 `.md`。

示例：

- `spec-登录-流程.md`
- `design-背包系统-重构.md`
- `spec-login-flow.md`
- `design-inventory-refactor.md`

### 扩展词汇

现有词汇无法准确表达时可新增，须同时提供中英文对照并登记到上表中。不为一次性表述造词。

## 语言模式

两种模式，同一目录内保持一致：

- **中文模式**：三个字段均用中文。`主题` 多词不加分隔符（如 `背包系统`）。
- **英文模式**：三个字段均用英文。`类型`、`行为` 取受控英文词汇，`主题` 转为英文 slug（小写，多词以 `-` 连接）。

两种模式均以 `-` 连接三个字段。

## 解析规则

`主题` 内部可能含 `-`。解析顺序：

1. 去掉 `.md`。
2. 从左匹配受控词汇表，切出第一个已知 `类型`。
3. 从右匹配受控词汇表，切出第一个已知 `行为`。
4. 中间剩余部分为 `主题`。

正常命名下 `类型`、`行为` 不会出现在 `主题` 首尾，此规则自然成立。

## 约束规则

- 禁止空格、路径分隔符（`/`、`\`）、换行、tab 及 `:` `*` `?` `"` `<` `>` `|` `%` `#` `&`。
- 英文字段统一小写。
- 总长度（含 `.md`）≤ **100 字符**。
- 禁止 `副本`、`copy`、`new`、`final`、`最新版`、`(1)` 等临时后缀。

## 重名处理

- 无同名文件 → 直接创建。
- 有同名文件 → 先判断是"更新"还是"新增并列"，不自动覆盖或随机命名。
- 同主题、同行为、内容持续演进 → 原地更新。
- 需要长期并存时，才新增并列文档。常见维度：职责侧、读者、平台、版本、阶段、方案。
- 并列文档优先在 `主题` 中加区分信息，`主题` 已稳定时才放 `行为` 中。
- 区分信息用稳定限定词：`客户端` / `client`、`服务端` / `server`、`编辑器` / `editor`、`V2`、`预研` / `draft`、`正式版` / `final`。不用时间戳或随机编号。

示例：

- `spec-登录V2-流程.md`
- `spec-登录服务端-流程.md`
- `design-背包系统正式版-重构.md`
- `spec-login-v2-flow.md`
- `design-inventory-server-refactor.md`

## 避免同义重复

`类型` 和 `行为` 含义重叠时选不同的词：

- 避免：`design-背包系统-设计.md`
- 改为：`design-背包系统-重构.md`

少数情况确有合理重叠（如"设计文档的设计过程"），但绝大多数情况下视为命名失误。

## 迁移规则

- 新建文档须遵守本规范。
- 旧文件名不合规时，原地重命名收敛，不新建合规副本。
- 内容更新、重写、拆分、合并、归档时，须按本规范重命名。
- 仅移动目录或调整分类、且旧名已合规的，可不改名。
- 改名后须同步更新所有引用该文件的链接或索引。
