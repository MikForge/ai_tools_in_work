# 知识库读取

## 概述

配置驱动的只读 skill，从项目知识库加载文档。所有路径和分类均来源于 `.knowledge-base.yml`。仅通过索引导航 — 永远不要直接扫描目录。

## 原则

1. **配置是唯一真相源。** 首先读取 `.knowledge-base.yml`。如果不存在，停止并告知用户运行 `knowledge-base-update` 初始化。
2. **仅通过索引导航。** 文档发现必须经过 README.md 索引。禁止在 KB 根目录使用 `ls`、`find`、`grep` 或任何目录扫描。
3. **自顶向下，禁止跳级。** 根索引 → 分类索引 → 文档，每一步依赖上一级的索引。
4. **知识库边界。** 只搜索 `.knowledge-base.yml` 中定义的 `root` 路径内的文档。KB 根目录外的文档不在范围内。
5. **诚实面对空结果。** 分类为空或无匹配 → 如实报告。禁止编造内容或到 KB 根目录外搜索。

## 模式

| 模式 | 触发条件 | 行为 |
|------|---------|------|
| **搜索** | 提供了 `query` 参数 | 匹配分类 → 读取分类索引 → 匹配文档 → 返回结果 |
| **浏览** | 无 `query`（直接调用） | 从根 README 开始 → 用户选层 → 用户选分类 → 用户选文档 |

## 工作流

### 搜索模式

1. **读取配置** — 解析 `.knowledge-base.yml`，获取 `root` 和 `categories`。
2. **读取根索引** — `{root}/README.md`，获取分类概览。
3. **匹配分类** — 将 `query` 与 `categories[].name` 和 `description` 进行匹配。若匹配到多个分类，列出候选项让用户选择。若显式提供了 `category` 参数则跳过此步。
4. **读取分类索引** — `{root}/{category.path}/README.md`，获取文档列表。
5. **匹配文档** — 在分类索引中通过标题和摘要匹配查询。
6. **返回结果** — 前 1-3 篇全文加载，其余仅返回标题和摘要。无匹配时如实报告"未找到"。

### 浏览模式

1. **读取配置** — 同搜索模式。
2. **展示根索引** — 从 `{root}/README.md` 展示层级和分类。
3. **逐层导航** — 每层让用户选择（层 → 分类 → 文档）。禁止一次性输出整个目录树。

## 输出格式

```markdown
## 知识库结果："<query>"

### 匹配文档

- **[文档标题.md]({root}/{category}/文档标题.md)** — {分类名}
  {完整内容}

### 其他匹配

- **[其他.md]({root}/{category}/其他.md)** — {一句话摘要}
```

规则：
- 最多 3 篇全文加载，其余仅返回标题和一句话摘要。
- 每篇标注分类名和相对路径。
- 无结果：`知识库中未找到与 "<query>" 匹配的文档。`
- 分类为空：`分类 "<name>" 存在，但尚无文档。`

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 使用 `ls`/`find`/`grep` 发现文档 | 仅通过 README.md 索引导航 |
| 跳过 `.knowledge-base.yml` 猜路径 | 始终先读取配置 |
| 浏览模式一次性输出整个目录树 | 逐层展示，等待用户选择 |
| 搜索 KB 根目录之外的文件 | 只读取配置中 `root` 路径下的文件 |
| 一次加载 5 篇以上文档到上下文 | 最多 3 篇全文，其余仅摘要 |
| 无匹配时编造内容 | 如实报告"未找到匹配文档" |
| 查询匹配多个分类时自行决定 | 列出候选分类，询问用户选择 |
