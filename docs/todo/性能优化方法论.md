# 游戏前端性能优化方法论

## 动机 / 背景

游戏前端性能问题通常分散在 CPU、GPU、Draw Call、GC、加载等几个核心方向，每个方向的分析链路和优化手段差异很大。需要一个按方向梳理的方法论框架，排查时知道先看哪个指标、用什么工具、上什么手段。

## 大致思路

| 方向 | 关注指标 | 典型优化手段 |
|---|---|---|
| CPU | 帧耗时分布、脚本耗时、逻辑热点 | 分帧执行、LOD 逻辑降频、对象池、空间换时间 |
| GPU | 填充率、带宽、shader 复杂度 | 降分辨率/半透明混合裁剪、简化 shader、剔除 overdraw |
| Draw Call | DC 数量、批次合并率 | 静态/动态合批、GPU Instancing、SRP Batcher、图集 |
| GC | 暂停帧、分配频率、堆大小 | 避免装箱、对象池、struct 替换 class、增量 GC |
| 内存 | 驻留集、纹理/网格占用、碎片 | 资源生命周期管理、按需加载/卸载、压缩格式 |
| 加载 | 加载耗时、IO 吞吐、卡顿帧 | 异步/后台加载、AssetBundle 拆分、预加载、流式加载 |

## 关键约束 / 注意点

- 先定位瓶颈在哪个方向（profiler 确认），不要凭直觉优化
- 各方向存在 trade-off：合批可能增加内存、降 shader 复杂度可能影响画面
- 移动端优先关注发热和功耗，仅帧率达标不够
- 优化粒度：先砍大的（DC 从几百降到几十），再抠细节
