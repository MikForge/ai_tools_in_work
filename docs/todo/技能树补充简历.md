# 技能树补充简历

## 动机 / 背景

现有 resume-tailor / tailored-resume-generator 依赖 JD + 现有简历做匹配，但用户往往说不清楚自己会什么、或者简历遗漏了隐性技能。用"技能树"这种结构化输入方式，让用户像填技能树一样逐层勾选自己的技术栈和软技能，再由 agent 自动注入到简历中，既降低输入门槛，也减少遗漏。

## 大致思路

- 定义一份分层级的技能树模板（大类 → 子类 → 具体技能），覆盖常见岗位方向
- 用户在技能树上勾选掌握程度（熟练/了解/精通）
- Agent 读取勾选的技能树，对照 JD 提取匹配项，自动生成简历中的"技能"section 和"项目经验"中的技术关键词
- 支持自定义添加技能树中没有的技能

## 关键约束 / 注意点

- 技能树模板需要覆盖主流岗位（前端/后端/数据/DevOps/产品等），但不宜过大导致选择疲劳
- 掌握程度要如实反映，不能夸大（和现有 skill 的 truthfulness 原则一致）
- 可考虑分层展示：先选大类，再展开子类，避免一次性呈现几百项
