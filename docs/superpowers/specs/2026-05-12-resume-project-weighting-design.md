# 简历项目结构重构 Spec

## 背景

`job/简历-完整版.md` 面向游戏客户端 / Cocos Creator / H5 小游戏岗位。上一版尝试将广州A公司经历写成“项目群 + 能力模块”，能解决《求生王》5 个月项目过重的问题，但直接放进简历后阅读感偏技术总结，不够像项目履历。

简历仍应按项目呈现，方便 HR 和面试官按项目追问；同时可以用广州A公司的真实工作周期承接多个代表项目，避免让《求生王》单独承担 6 年经验的证明压力。

## 目标

将项目经验调整为：

> 公司阶段总标题 + 代表项目子标题

既保留项目边界，又用真实公司周期说明这些项目都属于同一段工作经历。

## 非目标

- 不把《求生王》的 5 个月周期伪装成多年项目。
- 不将项目经验写成“长线运营与玩法系统 / 工具链 / 性能优化”这种文章式能力模块。
- 不编造上线结果、用户规模、收入、团队规模或量化指标。
- 不删除 Cocos Creator 框架能力，只降低它的独立权重。

## 推荐结构

### 1. 广州A公司项目经历（2020.09 - 2025.08）

作为公司阶段总标题，下面按代表项目展开。

#### 海外 H5 长线运营项目（2021.09 - 2025.08）

第一权重项目，证明真实商业项目、长线运营、多玩法、多语言、工具链和性能优化。

建议保留 8-9 条，覆盖：

- 家园放置、寻物、物品归类、运营活动。
- A\* 寻路、FSM、地图滑动、不规则命中检测。
- 对象池、资源预加载、GC 控制。
- 关卡编辑器、配置表系统、图集打包工具。
- 多语言、本地化、活动数据结构。

#### 《求生王》Cocos Creator 框架预研（2025.03 - 2025.08）

第二权重项目，作为近期 Cocos Creator 框架能力展示。

建议保留 5-6 条，表达为“预研 / 项目骨架 / 基础能力梳理”，覆盖：

- PureMVC 分层。
- 启动流程状态机。
- UI 管理。
- 资源管理。
- Protobuf + WebSocket 网络层。
- Cocos 热更新。

关键措辞：

> 设计启动流程状态机，梳理框架初始化、热更新、SDK、登录鉴权、长连接、数据拉取与进游戏流程，明确各阶段初始化顺序与异常处理边界。

不要写“10 状态”或过强的长期验证表达。

#### 《爱江山》女性向换装 / 化妆休闲项目（2020.09 - 2021.09）

第三权重项目，补充表现系统、动画、状态机和活动主题扩展能力。

建议保留 5-6 条，覆盖：

- 角色分层渲染。
- 遮罩、Tween、DragonBones。
- FSM 管理换装 / 化妆状态。
- 主题替换系统。
- 任务、排行榜与运营活动结构。

### 2. 广州B公司 MMORPG H5 商业项目（2018.06 - 2020.06）

独立保留，作为早期 UI 架构、通信和 SDK 接入能力证明。

建议保留 4-5 条，覆盖：

- MVC 界面层。
- 弹窗、列表、多窗口栈。
- 协议解析、错误码处理、数据一致性。
- 统一 SDK 封装层。

## 顶部模块同步

「核心优势」保留“长期沉淀 + 近期迁移”的叙事，但措辞不要替代项目经历本身：

1. 在广州A公司多款 H5 / 小游戏项目中参与多年持续迭代，覆盖多玩法、运营活动、多语言 / 本地化、性能优化和线上问题处理。
2. 自研关卡编辑器、配置生成、图集打包等工具链，提升内容生产与资源构建效率。
3. 近期完成 Cocos Creator 客户端框架预研，将长期项目中沉淀的 UI、资源、网络、状态机等能力迁移到新项目骨架。
4. 连续两年获评公司年度优秀员工。

## 验收标准

- 项目经验按项目走，而不是按能力模块写。
- 广州A项目经验使用公司总标题承接真实工作周期。
- 广州A下包含海外 H5、《求生王》、《爱江山》三个代表项目。
- 《求生王》保留 2025.03 - 2025.08，不暗示多年线上验证。
- 不出现 `10 状态`、`MacroCommand`、残缺标题或文章式能力模块标题。
- 简历整体仍保持游戏客户端架构 / 工程化能力主线。
