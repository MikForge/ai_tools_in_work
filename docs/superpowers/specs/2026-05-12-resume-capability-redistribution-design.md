# 简历能力分布重构 Spec

## 背景

当前 `job/简历-完整版.md` 已经回到按项目呈现的结构：

- 广州A公司项目经历
  - 海外 H5 长线运营项目
  - 《求生王》Cocos Creator 框架预研
  - 《爱江山》女性向换装 / 化妆休闲项目
- 广州B公司 MMORPG H5 商业项目

结构已经像正式简历，但《求生王》仍承担了较多“架构能力展示”。考虑到《求生王》周期为 2025.03 - 2025.08，约 5 个月，更可信的叙事应是：

> 广州A公司的长期项目沉淀了 UI、状态机、对象池、资源加载、工具链、配置驱动等基础能力；《求生王》阶段将这些能力迁移到 Cocos Creator 项目骨架中。

## 目标

调整 A 公司各项目之间的能力分布：

- 海外 H5 长线项目承担“长期业务 + 工具链 + 性能优化 + 基础能力沉淀”的主要证明。
- 《爱江山》承担“表现系统 + 状态管理 + 活动主题扩展”的证明。
- 《求生王》承担“近期 Cocos Creator 框架预研 + 能力迁移应用”的证明。

这样能减少《求生王》5 个月项目被过度追问的风险，同时保持客户端架构 / 工程化能力主线。

## 非目标

- 不把 Cocos Creator 专属内容写到 Egret 项目中。
- 不把《求生王》的 PureMVC、Cocos 热更新、Cocos bundle 管理直接挪到海外 H5 或《爱江山》。
- 不编造跨项目复用结果、上线效果、团队规模或量化指标。
- 不改变“按项目走”的简历结构。

## 可分配能力

### 可前置到广州A工作经历

适合放在工作经历中的跨项目能力：

- UI 组件与基础类库。
- 状态机。
- 对象池。
- 资源加载 / 预加载。
- 事件管理。
- 活动数据结构。

推荐表达：

```markdown
- 在多项目迭代中沉淀 UI 组件、状态机、对象池、资源加载、事件管理等基础能力，并在后续 Cocos Creator 预研项目中复用到新项目骨架。
```

### 可加强到海外 H5 长线项目

海外 H5 更适合承担“长期沉淀”的证明。

推荐新增或替换一条：

```markdown
- 在长期迭代中沉淀场景状态管理、资源预加载、对象池和活动配置结构，减少玩法与活动模块的重复实现。
```

也可保留已有工具链表达：

- 关卡编辑器。
- 配置表系统。
- TypeScript Interface 自动生成。
- 图集打包工具。
- 多语言管理。

### 可加强到《爱江山》

《爱江山》适合承担表现系统的结构化能力。

推荐新增或替换一条：

```markdown
- 将角色表现拆分为状态控制、动画编排、资源组合和妆容渲染多层逻辑，降低换装、化妆与活动主题扩展之间的耦合。
```

## 《求生王》保留内容

《求生王》应保留 Cocos Creator 专属和预研项目专属内容，建议压缩为 5 条左右：

- PureMVC 三层架构。
- 启动流程状态机。
- UI 管理系统。
- 资源管理与 Protobuf + WebSocket 网络层。
- Cocos 热更新。

建议将资源和网络合并表达，降低条目密度：

```markdown
- 封装资源管理器与 Protobuf + WebSocket 网络层，支持资源引用计数、加载竞态保护、心跳检测、超时处理和重连恢复。
```

## 推荐改动

1. 广州A工作经历替换“主导或参与客户端基础能力建设”那条，使其显式表达多项目沉淀与后续迁移。
2. 海外 H5 项目新增一条长期沉淀表达，或替换对象池条目为更综合的基础能力沉淀。
3. 《爱江山》新增一条表现系统拆分表达，或替换“实现妆容区域的精确渲染”那条。
4. 《求生王》合并资源管理和网络层两条，保留 Cocos 专属价值但降低篇幅。

## 验收标准

- 简历仍按项目呈现。
- 《求生王》条目数量减少或密度降低。
- 海外 H5 项目能体现长期沉淀的基础能力。
- 《爱江山》能体现表现系统拆分和状态管理。
- 不出现把 Cocos 专属能力写进 Egret 项目的表达。
- 不出现 `10 状态`、`MacroCommand` 等过强细节词。
