## 🧩 项目经验

### 《求生王》—— Cocos Creator 预研项目（2025.03 - 2025.08）

**技术栈：** Cocos Creator + TypeScript  
**项目角色：** 前端框架负责人

新项目预研阶段，从零搭建前端框架与底层基础能力，交付后供团队后续项目复用。

- 采用 PureMVC 三层架构，划分核心层与业务层并强制单向依赖，通过 Facade 统一管理 Command、Proxy、Mediator 的注册与通信。
- 设计 10 状态启动状态机，编排框架初始化、热更新、SDK、登录鉴权、长连接、数据拉取至进游戏的完整流程。
- 搭建 UI 管理系统：装饰器自动注册视图与中介者，配置注册器统一管理多层级 UI 与栈/缓存池策略。
- UI 基类提供预加载、初始化、显示、隐藏、刷新、销毁完整生命周期；容器组件自动发现节点组件并生成类型化绑定，消除手动属性声明。
- 封装资源管理器，按 bundle 加载与卸载，引入引用计数、预加载策略与加载竞态保护。
- 搭建网络层：Protobuf + WebSocket 长连接，4 状态连接状态机，心跳与长静默双重超时检测，指数退避重连。
- 消息管道基于有界队列与帧分发防止单帧消息堆积；HTTP 客户端支持拦截器链与失败重试。
- 接入热更新能力，封装版本校验、增量下载、失败重试流程，支持灰度发布与断点续传。
- 封装通用状态机框架、对象池、事件管理器、数学工具等基础模块，供启动流程、网络层、游戏逻辑多处复用。

### 海外 H5 项目（家园 + 寻物 + 运营系统）（2021.09 - 2025.08）

**技术栈：** Egret + TypeScript  
**项目角色：** 前端开发工程师

面向海外市场的长线运营休闲项目，涵盖家园放置、寻物、物品归类及复杂运营活动系统。

- 家园放置玩法中，基于 A\* 算法实现人物智能寻路与障碍绕行，引入 FSM 管理多个游戏状态，保证场景与 UI 切换的稳定性。
- 封装地图滑动交互组件，处理坐标映射、边界控制、惯性滑动与回弹等逻辑，优化大地图拖动体验。
- 自研多边形顶点编辑工具，支持策划配置默认摆放方案与复杂点击区域。
- 寻物 / 物品归类玩法中，使用向量叉乘实现多边形命中检测，支持不规则物体点击与拖拽；以配置驱动方式封装拖拽、吸附、结算流程，通过配置即可扩展新关卡。
- 使用对象池管理高频节点，配合资源动态预加载与按需释放，有效降低运行时 GC 抖动，优化内存占用。
- 提供关卡编辑工具，可视化配置物体位置、多边形顶点与分类规则，显著提升关卡配置效率。
- 搭建多语言管理器与配置表系统：基于 JSON 的多语言管理，Python 脚本自动生成 TypeScript Interface，降低人工维护成本。
- 开发命令行图集打包工具，统一资源构建流程，减少手动操作与出错率。
